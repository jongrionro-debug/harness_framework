# Stitch 전달용 프롬프트 초안

## 사용 방법

아래 프롬프트는 Stitch에 그대로 붙여 넣을 수 있는 초안이다.

실제 전달할 때는 아래 두 가지만 추가하면 된다.

1. 현재 화면 스크린샷 첨부
2. 특히 바꾸고 싶은 화면 우선순위 2~3개 표시

---

## Prompt

```txt
우리 프로젝트의 현재 UI를 기준으로 재디자인해 주세요. 완전히 다른 제품을 새로 상상하는 것이 아니라, 현재 구조와 역할 분리를 유지한 채 정보 우선순위, 화면 밀도, 탐색 흐름, 상태 표현을 더 좋아지게 만드는 방향을 원합니다.

프로젝트 요약:
- 이 프로젝트는 지역 교육 프로그램 운영 시스템입니다.
- 한 기관 안에서 운영자와 강사가 함께 사용하는 웹 앱입니다.
- 운영자는 기관 생성, 운영 기본정보 설정, 사용자 초대/승인/배정, 세션 생성, 제출 기록 조회를 담당합니다.
- 강사는 자기에게 배정된 세션만 열어서 출석, 교육일지, 첨부 문서를 제출합니다.
- 핵심 단위는 "한 날짜의 한 수업 세션"이며, 운영자가 세션을 먼저 만들고 강사가 그 세션 기록을 제출하는 구조입니다.

반드시 이해해야 하는 제품 구조:
- 운영자와 강사는 같은 인증 시스템을 쓰지만 진입 화면이 다릅니다.
- 운영자 주요 화면:
  - /dashboard
  - /settings
  - /users
  - /records
- 강사 주요 화면:
  - /sessions
  - /sessions/[sessionId]
- 온보딩 관련 화면:
  - /login
  - /signup
  - /organization
  - /approval-pending

현재 우리가 원하는 시각적 방향:
- 따뜻하고 친근한 메신저형 운영 도구
- 밝은 warm neutral 배경
- 빠르게 스캔 가능한 구조
- 차갑고 무거운 백오피스 느낌은 피하기
- 카카오톡 계열처럼 빠르고 익숙한 사용감은 참고할 수 있지만 그대로 베끼지는 않기

시각 가이드:
- Primary Accent: #FEE500
- Accent Surface: #FFF3A4
- Background: #F6F1E8
- Surface: #FFFDF8
- Surface Alt: #F3EBDD
- Border: #E5DAC8
- Text Primary: #1C1A17
- Text Secondary: #6B665E

반드시 피해야 하는 것:
- 보라/인디고 중심의 전형적인 SaaS 대시보드
- 과한 글래스모피즘, blur, neon glow
- 모든 섹션이 똑같은 카드 반복으로 보이는 구조
- 장식 위주 배경과 랜딩페이지 같은 과한 연출

디자인 목표:
- 운영자 대시보드는 "다음 할 일"과 "현재 운영 상태"가 더 한눈에 보여야 합니다.
- 강사 세션 상세 화면은 더 짧고 빠르게 입력할 수 있어야 합니다.
- 기록 브라우저는 검색/필터/상세 진입 흐름이 더 명확해야 합니다.
- 설정 화면과 사용자 관리 화면은 정보가 많아도 답답하지 않게 재구성되면 좋겠습니다.
- 현재 제품의 역할 기반 구조는 유지하되, 정보 계층과 시각 밀도를 더 잘 설계해 주세요.

레이아웃 기대:
- 데스크톱 우선
- 모바일에서도 무너지지 않는 구조
- 좌측 탐색, 상단 필터, 리스트 + 상세 패널, 버블형 구획 등은 적극적으로 활용 가능
- 화면마다 모두 같은 패턴을 반복하지 말고, 역할에 따라 시각 리듬을 달리해 주세요

출력 기대:
- 화면별 재디자인 방향 제안
- 특히 /dashboard, /sessions/[sessionId], /records 에 대한 우선 개선안
- 가능하면 현재 UI와 비교했을 때 어떤 점이 좋아졌는지 설명
- 컴포넌트/섹션 단위로 구조를 제안

참고 자료:
- 첨부한 현재 UI 스크린샷
- SYSTEM_REPORT.md: 제품 구조와 구현 범위 설명
- PRD.md: 제품 요구사항
- UI_GUIDE.md: 시각 언어와 금지 패턴

중요:
- 현재 UI를 모르는 상태에서 새 제품을 상상하지 말고, 첨부한 스크린샷을 기준으로 개선해 주세요.
- 기능 범위는 유지하고, 주로 UX 구조와 시각 설계를 재정리해 주세요.
```

## 짧은 버전

시간이 없을 때는 아래 축약본을 쓸 수 있다.

```txt
첨부한 현재 UI 스크린샷을 기준으로 우리 프로젝트 UI를 재디자인해 주세요. 이 제품은 지역 교육 프로그램 운영 시스템으로, 운영자와 강사가 같은 인증 시스템을 쓰지만 화면 역할이 분리되어 있습니다. 운영자는 기관 생성, 설정, 사용자 관리, 세션 생성, 기록 조회를 담당하고, 강사는 배정된 세션의 출석/교육일지/첨부를 제출합니다. 따뜻한 메신저형 운영툴 톤과 노란 accent 기반 브랜드 방향은 유지하고, 전형적인 보라색 SaaS 대시보드 느낌은 피해주세요. 특히 /dashboard, /sessions/[sessionId], /records의 정보 우선순위와 화면 밀도를 더 좋아지게 재구성해 주세요.
```

## 화면 우선순위 예시

Stitch에 아래 한 줄을 추가하면 결과가 더 선명해진다.

```txt
이번 라운드에서는 /dashboard, /sessions/[sessionId], /records를 우선 재설계하고, 나머지 화면은 같은 톤으로 확장 가능한 시스템처럼 제안해 주세요.
```

## 유지 / 변경 예시

필요하면 아래도 함께 붙이면 좋다.

유지하고 싶은 것

- 운영자와 강사의 역할 분리
- warm neutral 배경과 노란 accent
- 친근한 운영툴 톤

개선하고 싶은 것

- 카드 반복형 구조를 줄이고 정보 계층을 더 분명히 만들기
- 강사 입력 화면에서 주요 액션을 더 빨리 찾게 만들기
- 운영자 대시보드에서 다음 행동을 더 선명하게 보이게 만들기
- 기록 브라우저의 필터와 탐색 흐름 개선
